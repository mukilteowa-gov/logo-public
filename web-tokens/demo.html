02

Accessibility

This guide and any work derived from it must meet WCAG 2.1 AA at minimum.

Foundational principle

Accessibility takes priority over brand rules.

If a design meets the brand guide but fails WCAG 2.1 AA, the design must be corrected. This applies to backgrounds, type sizing, contrast, focus indicators, and keyboard operability.

If the guide is silent, follow WCAG. If they agree, follow both. If they conflict, follow WCAG and notify Communications so the guide can be amended.

Requirements that apply across every section:

  • Text contrast meets 4.5:1 (normal text) or 3:1 (large text and UI component boundaries).
  • Every interactive element is keyboard operable.
  • Every form input has a programmatically associated label.
  • Focus is visible on all focusable elements.
  • Landmarks use semantic elements: <header>, <nav>, <main>, <footer>.
  • Motion honors prefers-reduced-motion.
  • The page declares a language on <html>.

05

Web surface interpretation

Web surfaces are not pure white or pure black. This section defines how the approved backgrounds in §03 apply to web environments, in measurable terms.

From the guide "Do not place the logo on any low contrast background. Do not use the logo with any unauthorized background color."

1 · Replace named colors with roles

Guide termWeb role
WhiteLight neutral surface
BlackDark neutral surface
Deep navyUnchanged — specific brand color
AquaUnchanged — specific brand color

2 · Constrain what counts as each role

A surface is neutral only if it meets both conditions:

  • Luminance falls within the range below.
  • RGB values are close enough to appear gray — no visible tint.
RoleRelative luminanceMax−min RGB channel delta
Light neutral≥ 0.85≤ 12
Dark neutral≤ 0.10≤ 12

Luminance follows the WCAG 2.1 relative-luminance formula.

Channel delta is a local guardrail. It is not part of WCAG. The threshold may be adjusted in future revisions.

3 · Contrast

Logo contrast must meet WCAG 2.1 AA:

  • 3:1 for the logomark (SC 1.4.11)
  • 4.5:1 for the "CITY OF MUKILTEO" logotype (SC 1.4.3)

AAA (7:1) is preferred.

4 · Prohibitions still apply

These rules still apply:

  • Saturated color backgrounds (pink, lime, red, tinted blues — anything that fails the channel-delta test)
  • Textured or busy backgrounds (photos, patterns, illustrations)
  • Altered, recolored, or substituted logo colors
  • Clear-zone violations — type, imagery, or shapes inside the zone
  • Low-contrast placements that fail §3, regardless of §2

5 · Tier structure

TierSurfacesWhen
Preferred #ffffff · #000000 · deep navy #003660 · aqua #4F9B99 Always approved. Default for anything brand-facing.
Acceptable Any light or dark neutral that passes both tests in §2 and contrast in §3. When a preferred surface is impractical — OS dark mode, soft off-white UI, print on uncoated stock.
Prohibited Saturated, tinted, textured, or low-contrast backgrounds. Never, without explicit permission from the communications office.

Use the tokens · downloads

The authoritative reference is the brand guide PDF. All files below live in the public repo at github.com/mukilteowa-gov/logo-public — pull them directly, or vendor them into your project.

Integration — three-line drop-in

These three stylesheets load the brand palette, brand font stacks, and the free Google Fonts substitutes they fall through to. Add them to <head>:

<!-- Option 1: vendored into your project -->
<link rel="stylesheet" href="/path/to/web-tokens/colors.css">
<link rel="stylesheet" href="/path/to/web-tokens/fonts.css">
<link rel="stylesheet" href="/path/to/web-tokens/fonts-google.css">

<!-- Option 2: pulled directly from GitHub via jsDelivr CDN -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/mukilteowa-gov/logo-public@main/web-tokens/colors.css">
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/mukilteowa-gov/logo-public@main/web-tokens/fonts.css">
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/mukilteowa-gov/logo-public@main/web-tokens/fonts-google.css">

Then in your CSS:

body { font-family: var(--mukilteo-sans); color: var(--mukilteo-deep-navy); }
h1   { font-family: var(--mukilteo-sans-condensed); }
.btn { background: var(--mukilteo-aqua); color: #fff; }

For standalone robustness, hard-code the primary colors and a system font stack as a fallback block before the <link> tags — see the <style id="fallback-tokens"> at the top of this page's source for a ready-to-copy example.
